---
title: Subscriber Management Process
---

This sequence diagram represents the following steps:

1. The user requests access to manage their subscriptions.
1. The Authentication Service verifies the user's permissions through the database.
1. If the user has necessary permissions, they can interact with the Subscriber Management Service to perform actions like subscribing, unsubscribing, or viewing their subscriptions.
1. Depending on the user's action, the Subscriber Management Service communicates with the database to execute the request and provides feedback to the user.
1. If the user does not have the required permissions, access to subscriber management is denied.

      ```mermaid
      sequenceDiagram
        participant U as User
        participant SM as Subscriber Management Service
        participant DB as Database
        participant AU as Authentication Service

        U->>AU: Requests access to subscriber management
        AU->>DB: Verify user permissions
        DB-->>AU: Permissions result
        alt If User has Permissions
            AU-->>U: Access Granted
            U->>SM: Request to subscribe/unsubscribe/view subscriptions
            alt If subscribing/unsubscribing
                SM->>DB: Update subscription status
                DB-->>SM: Confirm subscription status update
                SM-->>U: Subscription status update confirmation
            else If viewing subscriptions
                SM->>DB: Retrieve subscription details
                DB-->>SM: Subscription details
                SM-->>U: Display subscription details
            end
        else If User lacks Permissions
            AU-->>U: Access Denied
        end
      ```
