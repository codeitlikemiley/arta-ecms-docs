### Subscribers Service

#### Summary

The Subscribers Service is designed to manage user subscriptions within the system. It allows users to subscribe to updates on various entities like issues or topics, thereby staying informed about changes or new information.

#### Endpoints

1. **AddSubscription**
    - **Purpose**: Subscribe a user to an entity for updates.
    - **Request Parameters**:
        - `user_id` (string): The ID of the user subscribing.
        - `entity_id` (string): The ID of the entity to subscribe to.
    - **Response**:
        - `success` (Boolean): Indicates if the subscription was successful.

2. **GetSubscriptions**
    - **Purpose**: Retrieve a list of all subscriptions for a user.
    - **Request Parameters**:
        - `user_id` (string): The ID of the user whose subscriptions are being requested.
    - **Response**:
        - `entity_ids` (string array): List of IDs of the entities the user is subscribed to.

3. **RemoveSubscription**
    - **Purpose**: Remove a user's subscription to an entity.
    - **Request Parameters**:
        - `user_id` (string): The ID of the user.
        - `entity_id` (string): The ID of the entity to unsubscribe from.
    - **Response**:
        - `success` (Boolean): Indicates if the unsubscription was successful.
